HOTAS and MFD Maps (HOTAS Warthog)

The following sections provide graphical representations of the Warthog HOTAS and Cougar MFDs for each aircraft module.  Most of these images are scaled down to fit the browser displayscaled down to fit the browser display.  As such, you are encouraged to download these images from within this browser to a folder of your choice (simply right click on the image and select "Save Image As," or you can navigate to Docs\Profile JPGs to access the originals.  You are also encouraged to print out the images, especially the MFD inserts..

The MFD pages should be sized for the overlay insert slots in the Cougar MFDs.  Be sure to print out the page with 100% scaling, then trim to fit.  Additionally, high visibility JPEGs of the MFD inserts are available for some of the modules.  These inserts are intended for people who are colorblind, fly DCS in a dark area, or just don’t want to deal with a stylistic MFD insert.  They can be found in the JPG folder under the folder Docs\Profile JPGs\_High Contrast MFDs. 

Important: Be sure you load the diff.LUA files for both your HOTAS and the keyboard prior to using this profile in-game.  See the section titled Controller Input Setup in the Setup Guide PDF for further instructions.

If you have any questions, comments or suggestions, please feel free to PM “Home Fries” at CombatSim and Eagle Dynamics forums, or “HomeFries” at Combat Ace and SimHQ.


Nomenclature

Before describing the following routines, naming conventions need to be addressed.  For starters, “shift” or “shifted state” refers to the S3 button being held, and corresponds to the /I state in TARGET/Foxy programming.  States referring to the Boat Switch (Warthog) are /U for the Up/back position, /D for Down/forward, and /M for Middle.  This allows six possible assignments for each button, and combinations will be compound references (e.g. /UM for both Boat Up and Middle states and /DI for S3 in and Boat Switch down).  All buttons will be referred to using standard TARGET/Foxy terminology.

MFDs buttons are referred to as OSB#, and cycle clockwise (e.g. OSB1 is the left-most button on the top row, counts to OSB5 to the right across the top, down the right side to OSB10, left across the bottom to OSB15, and finally up the left column to OSB20).  MFD LEDs are LED1 and LED2 (left to right).   See the MFD graphical inserts for special instructions and exceptions.

On HOTAS graphics, Gear State is represented for GSU/GSD for logical Gear State Up/Down respectively; see Gear, Flaps, Rudder and Brakes (with LED Indications) for more details.

DirectX outputs will be referred to as DX# (e.g. DX21) and reference the output seen by DCS (and not necessarily the controller button itself).  For example, TG2 normally sends the DX6 command, but if I were to use this for communications (as in the UH-1) I would map it to the DX21 output.  Therefore, DX6 doesn’t necessarily mean TG2, and DX21 doesn’t necessarily mean T3 or LDGH.

TEMPO Long commands are normally displayed in Italics on the control map graphics, with Red Italics denoting that the function can be activated either by TEMPO Long or by S3+TEMPO Short, /I Red Italics indicating that S3+TEMPO Long is required, and Italics sans special color to denote that the function can only be activated by TEMPO Long.  /I unitalicized indicates that the function isn't TEMPO'd (i.e. S3+button is required to activate the function).

SnapView Substitutions for VR

Since SnapViews have no use in VR, most SnapViews have been replaced with either Rudder Trim or VR Zoom and Spyglass Zoom views, depending on aircraft capability and SnapView orientation.  Unless otherwise specified on the HOTAS graphic itself, SnapViews will be remapped to the following functions as follows for any substituted SnapView.  Unshifted SnapViews will remap to unshifted functions, and shifted SnapViews will remap to shifted functions.  All functions are held (not toggled).  If an entire hat is remapped it will map as the table below.  Otherwise, individual axes may be remapped to rudder trim or zoom functionality based on orientation.

VR Zoom (Hold)
Rudder Trim Left Substitute SnapView Functions Rudder Trim Right
Spyglass Zoom (Hold)

Example 1: The JF-17 Thunder uses S3+H4 for SnapViews.  S3+H4U/D is used for VR Zoom and Spyglass Zoom respectively, and S3+H4L/R is used for Left and Right Rudder Trim.

Example 2: The MiG-15 uses H3 for SnapViews, so H3U/D is mapped to VR Zoom and Spyglass Zoom. Rudder trim is mapped to H3L/R.

Example 3:  The SA342 Gazelle uses S3+H3 for SnapViews, and does not use S3+H3U/D for gear/flap cycle with the HOTAS Cougar.  Therefore, S3+H3U/D is mapped to VR Zoom and Spyglass Zoom respectively.  Since the SA342 does not have conventional rudder trim, S3+H3L/R is not remapped.

MFD/Direct Entry

The 3:4 orientation of the JF-17 MFDs, along with the additional buttons for each MFD requires that TEMPO Long be utilized to command the extra buttons.  The top five vertical buttons on the main MFD display are actuated with OSB06-10 and OSB16-20, while the bottom three vertical buttons on the auxiliary MFD display are actuated with OSB08-10 Long and OSB16-18 Long.  The LMFD is the same for all users.  The JF-17 profile has different RMFD mapping based on the Number of Thrustmaster MFDs as set in CTS.  The unshifted RMFD buttons always actuate the in-game Right MFD.  For users with two MFDs (or three MFDs if the NS430 is enabled for the JF-17), holding S3 while commanding the same RMFD buttons will actuate the in-game Center MFD.

Each MFD mapping scheme requires a separate diff.lua file.  Be sure to choose the diff lua that best matches your MFD configuration.  For example, if you are running a HOTAS Warthog with two MFDs (or three MFDs with the NS430 enabled), choose TARGET JF17 WH2 v2.xx.diff.lua.  If you have a HOTAS Warthog with three or more MFDs, select TARGET JF17 WH3 v2.xx.diff.lua.

Note that MFDs 3 and 4 are separate controllers independent of the TARGET profile.  To enable the MFD3 functionality, import the TARGET JF17 MFD3 v2xx.diff.lua to the MFD3 or MFD4 controller in the DCS Options menu.  See the section in the Setup Guide PDF  titled Controller Input Setup for details.

Weapons Systems Officer (WSO)

Important: to use the full profile, you must set Joy_Btn25 and Joy_Btn26 as modifiers for the WSO station.

If you would like to use the WSO station and you don't have a separate left handed controller lying around, you may want to set Joy_Btn25 and Joy_Btn26 (the Dogfight/Boat Switch DirectX buttons) as modifiers in the Options menu for the WSO.  This is necessary to use a single controller to control both left and right hand controllers at the WSO station.  Boat Fwd (BSF) maps to the RHC, and Boat Aft (BSB) maps to the LHC. Center position is for flight controls, though most functions will default to the LHC.  Specific mappings are addressed in the User Reference.  If you do have an extra Cougar or Warthog stick lying around, you can set BSB and then use the secondary stick for the RHC, or conversely set BSF and use the secondary stick for the LHC.  The lua profiles TARGET F15EWSO L/RHC whc v2xx.diff.lua (for left and right controllers) have been included specifically for this use.  Mappings for the spare controllers are addressed in the User Reference.

Additional MFDs

The F-15E profile maps to all three MFDs and the UFC with two Thrustmaster MFDs, with both MFDs directly mapping to the left and right MPDs in the jet.  Alternate functions (S3 and TEMPO Long functions) provide mappings to the UFC (LMFD) and the MPCD (RMFD).  However, CTS nominally supports more than two MFD, even though additional MFDs are not part of the TARGET profile.

If the Number of Thrustmaster MFDs is set to more than 2 in CTS, then the script will assume that you want to use the third and perhaps fourth MFDs to control the MPCDs.  As such, the TEMPO functionality will be removed from the RMFD for the pilot station if at least three MFDs are present.  If you do have a third MFD present, you may import the TARGET F15EPlt MFD3 v2xx.diff.lua, to MFD3, and the functionality will default to unshifted for the MPCD and shifted for the additional pilot functions.  Finally, with four MFDs, one of the MFDs can be set to the UFC with TARGET F15EPlt UFC v2xx.diff.lua, negating the need for shifted commands on MFD1.  See the MFD Insert for the UFC in the next section under WSO configurations.
