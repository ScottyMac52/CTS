To update this script, you need only to run CTSupdate.exe or launch the updater from within CTS as circled in red below, then select Normal Update.  Once you load CTS, it will automatically import your existing settings into the new database.  Then simply select Build Script to compose your new script files.  You should periodically check for updates.

If your install becomes corrupted, you can run the Updater and select a Full Install.  The Updater will force-download all files to the latest version.

If you wish to do a "clean install" but still keep your settings, you can delete all of the DCS Script files except for CTS.dat and the DB folder.  Then simply run the Updater and select Full Install.  However, this should seldom be necessary.

If you encounter a bug you wish to report, see this section for instructions.


Module Version Table

This TARGET script is both frequently updated and requires additional files like Diff LUA control profiles to properly operate.  This section contains a table that indicates the latest versions of these files by Module.  Diff LUAs are control profiles that must be imported into DCS World (See the section in the Setup Guide PDF  titled Controller Input Setup for details), though this column also represents control bindings for supported simulations outside of the DCS World ecosystem (e.g. IL-2 Sturmovik).  External Mods are optional mods that can be added to DCS World to enhance the profile's capabilities (see Appendix B or C in the Setup Guide PDF for details on setting up JSGME or OvGME respectively).  Finally, the JPEGs are the controller maps available for both Cougar and Warthog setups, with an updated version showing a non-transparent change in the profile for that module.  Version numbers marked in bold have changed since the last incremental update.

Changes to Default Values

At times, it may be necessary to change a default value to fix a bug, improve a feature, or alleviate confusion.  When this happens, your existing setting may be forced to take on the new value until you manually change it.  However, forcing this change for the duration of updates defeats the purpose of having customizable settings, so a value that is forced to a default will only do so for a single update version.  The follow-on version will no longer force the default value.

As a result, people who miss a version or two may not realize that a value has been changed.  This table keeps track of any default values that change over time, as well as whether the updated value was forced and the database version in which it was changed (and possibly forced).  It is recommended that you periodically check this table and ensure that your values correspond with the updates.  The first column indicates which module you should select on the Values page, with global vlaues listed as "Global."

Bug Reporting and Tracking

With the release of version 2.20, CTS has transitioned to the CTS bug tracker for bug reporting and beta testing.  The bug tracker uses the Trello engine, but it is set up to allow anybody (even people without a Trello account) to submit bugs and review progress.  If you have a (free) Trello account, you can also make comments on existing reports, whether to provide amplifying information or confirm that you were able to duplicate the bug.

Submitting a Bug Report

The default way to submit a bug report is via email.  If you have a Trello account, you can submit bugs using your own Trello email and you will be identified as the originator.  If you do not have a Trello account, you can use the following email to submit bug reports:
ctsbug+ymwcmjq7n4mmc5ggmyw6@boards.trello.com

In order to properly troubleshoot the issue, I need information to evaluate the problem. When you submit a bug report, put the title of the bug as the heading of the email. In the email body, please provide as much detail about the bug as possible, including steps to replicate it if possible.  Also, please be sure your report includes all of the items in the checklist below:
  • CTS version
  • DCS World version
  • Windows version
  • Aircraft or Modules affected
  • Cougar or Warthog
  • MFDs (yes/no)
  • Head Tracking (e.g. TrackIR, VR, and if VR whether trim functionality is set to 0 or 1)
  • Communication Settings enabled:
    • VAICOM (yes/no)
    • SimpleRadio (yes/no)
    • CommState (yes/no)
    • PTT (Keyboard or DirectX) and  PTT Key if Keyboard (otherwise will assume Scroll Lock)
  • Do you use TeamSpeak 3?  If so, which version?
Note: only CTS v2.20 and later is supported.  If you wish to report a bug for the legacy version of CTS, you may do so, but I cannot promise that it will be addressed.
