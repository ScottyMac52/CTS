Adding Custom Views and SnapViews to DCS World

This script makes use of custom SnapViews for Direct Entry (e.g. A-10C CDU, Ka-50 Rubicon) as well as general use for switch manipulation.  While I attempted to keep the SnapViews as close to default as possible, we must use custom SnapViews in order to achieve the ideal solution.

The issue is compounded because there is a single SnapView LUA file for all DCS aircraft, so replacing a SnapViews.lua to add A-10C snapviews will overwrite any custom FC3 or Black Shark snap views you might have.  Snapviews are also optimized for certain resolutions or aspect ratios, so what works for 1920x1080 (16:9, for what these included views are optimized) may not work as well for a wider aspect ratio (e.g. triple screen).  Add Boresight Views  to the mix, and you have a different set of data, meaning that there is no universal plug-and-play solution.

Should you already have custom SnapViews, or if the SnapViewsDefault.lua changes in the next DCS World update, then the solution is likely manual insertion.  In order to do this, open the SnapViews.lua in a Unicode compatible text editor like Notepad++.  Manual insertion of SnapViews is the topic of the rest of this appendix.

Creating SnapViews.lua

If you don’t already have a SnapViews.lua file in your C:\Users\<username>\Saved Games\DCS\Config\View folder, then copy  the DCS World\Config\View\SnapViewsDefault.lua to your Saved Games\DCS\Config\View folder.  Then rename the new copy SnapViews.lua.


Manual adjustments for Aspect Ratios other than 16:9

Because the script author uses a 1920x1080 display, all SnapViews are optimized for the 16:9 aspect ratio.  For people with wider displays or multi-monitor setups, the SnapViews can be maintained simply by increasing the viewAngle value in each SnapView (and default view) so that the Field of View (FOV) is proportional to the desired aspect ratio using the following equation:

viewAngle = old viewAngle * Monitor Aspect Ratio * 0.5625
(Note: multiplying by 0.5625 is the same as dividing by 16/9)

For example, if using a monitor with 3440x1440 resolution, the calculation for an 85 degree FOV would be as follows:

viewAngle = 85 * (3440/1440) * 0.5625
viewAngle = 85 * 2.39 * 0.5625
viewAngle = 114.271875

Note: viewAngle should only be increased, and not decreased.  SnapViews are intended to maximize the zoom of a desired display or console while still getting all relevant information onscreen.  Decreasing viewAngle from the default value is likely to chop a desired section of the SnapView.  Similarly, any calculated viewAngle should be rounded up for the same reason.

Since adjusting amost every viewAngle in SnapViews.lua is an exhaustive process, it is recommended to back up SnapViews.lua and make manual edits when SnapViews are updated in CTS.  Check the SnapViews row in the Module Version Table after each CTS update to see if there are any changes to the SnapViews witth the latest version.  Then use a text editor like Notepad++ with the Compare plugin to see what has changed between the existing SnapViews.lua and the updated SnapViews.lua.

If you wish to use the Boresight Views, but need to manually enter them, then replace the default view ([13])entries in your Snapviews.lua with the entries in the following section.
